# 제이제이컴퍼니 앱 개발 원칙

## 핵심 모토
**단순 명확 정확 추적 + 사용자 편리성**
- UI/UX: 단순하게
- 입력/출력: 명확하게
- 데이터: 정확하고 추적 가능하게
- 사용자: 불편함 없이

---

## 데이터 무결성 원칙 (절대 규칙)

### 1. localStorage 키 — 추가만, 절대 변경/삭제 금지
```
현재 키 목록 (건드리면 안 됨)
stores_v3
priceHistory_v3
wastePriceHistory_v3
tx_손호영_v3
tx_박기빈_v3
pay_손호영_today
pay_박기빈_today
done_손호영_YYYY-MM-DD
done_박기빈_YYYY-MM-DD
skip_손호영_YYYY-MM-DD
skip_박기빈_YYYY-MM-DD
cargo_today_손호영
cargo_today_박기빈
checkin_log_손호영_YYYY-MM-DD
checkin_log_박기빈_YYYY-MM-DD
route_order_손호영
route_order_박기빈
last_date_손호영
last_date_박기빈
unsent_손호영
unsent_박기빈
accounts_v1
```
새 기능 추가 시 → 새 키 이름으로만 추가

### 2. GAS로 보내는 tx 필드 — 추가만, 절대 변경/삭제 금지
```
확정 필드 (건드리면 안 됨):
id, 날짜, 시간, 직원, 거래처, 유종, 품목상세,
통수, 판매단가, 출고가, 폐유kg, 폐유매입단가,
식유금액, 폐유매입금, 차감청구,
식유마진, 폐유수익, 총수익,
수금방법, 수금액, 미수, 비고
```
새 필드 추가 시 → 기본값 필수, 기존 GAS 코드 영향 없어야 함

### 3. GAS 시트 컬럼 — 순서/이름 변경 금지
컬럼 추가는 맨 뒤에만 가능

### 4. 미수금 연속성
- 월 바뀌어도 리셋 없음
- 앱 업데이트해도 리셋 없음
- 전체 시트 누적 차감청구 - 전체 입금 = 미수잔액

### 5. 버전 관리 규칙
```
현재 버전: v3
구조 변경 시만 v4로 올림
v4 올릴 때: 반드시 v3 → v4 마이그레이션 코드 포함
절대 기존 데이터 덮어쓰거나 삭제하지 않음
```

## 배포 전 체크리스트

### 앱(HTML) 배포 시
- [ ] localStorage 기존 키 이름 바뀐 게 없는가?
- [ ] GAS로 보내는 tx 필드 구조가 동일한가?
- [ ] safeFetch / 큐 시스템 그대로인가?
- [ ] 실제 폰에서 납품 저장 → 시트에 데이터 들어오는가?
- [ ] 미수 계산 결과가 이전과 동일한가?

### GAS 배포 시
- [ ] 기존 type 핸들러 제거된 게 없는가?
- [ ] 기존 시트 컬럼 위치/이름 바뀐 게 없는가?
- [ ] 새 컬럼은 맨 뒤에 추가했는가?
- [ ] 기존 데이터로 테스트했는가?

## 업데이트 단계별 원칙

```
1단계 올림 → 폰에서 테스트 → 시트 확인 → 이상없으면
2단계 올림 → 폰에서 테스트 → 시트 확인 → 이상없으면
...
```
한 단계에서 문제 생기면 그 단계만 롤백. 이전 단계 데이터는 안전.
